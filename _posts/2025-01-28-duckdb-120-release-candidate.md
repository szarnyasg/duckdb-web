For the past few feature releases, we have observed that some bugs will just not be detected by our automated testing, despite its already massive and constantly increasing scope. So we publish a new feature release, people start actually trying the new release, and only then they run into obscure bugs. We then proceed to fix them and quickly release a “.1” bugfix release.

For the next feature release 1.2.0 we are trying something different: today we are publishing a [release candidate](https://en.wikipedia.org/wiki/Software_release_life_cycle#Release_candidate). The main goal of the release candidate is to allow users to test their existing workflows.
